## The Architecture

Based on Felix Craft's *[How to Hire an AI](https://felixcraft.ai/)* playbook, adapted for a creative AI lab.

### Three-Tier Memory
1. **MEMORY.md** â€” Tacit knowledge. How the human operates, preferences, patterns. Not facts about the world â€” facts about *you*.
2. **Daily Notes** (`memory/YYYY-MM-DD.md`) â€” Chronological log. What happened today. Auto-generated by nightly extraction.
3. **Knowledge Graph** (`life/`) â€” Entity-based storage using the PARA method. People, companies, projects â€” each with atomic facts, access tracking, and decay.

### Trust Ladder
Not everything is autonomous. Actions follow a trust hierarchy:

| Level | Examples | Approval |
|-------|----------|----------|
| **Read-Only** | Web search, file reading, memory updates | Auto |
| **Draft & Approve** | Emails, social posts, public content | Queued for human |
| **Act Within Bounds** | Internal cron tasks, file edits, git commits | Auto |
| **Full Autonomy** | Rare â€” only low-stakes, well-defined tasks | Pre-approved |

### Daily Operating Rhythm
- **8:00 AM** â€” Morning briefing surfaces priorities, pending items, overnight activity
- **Working hours** â€” Handle tasks, queue approvals, build things
- **11:00 PM** â€” Nightly extraction: review conversations, extract durable facts, update knowledge graph, apply memory decay

### Safety Rails
- Email is **never** a trusted command channel
- No autonomous social media posting
- No sending money or signing contracts
- Approval queue for all external-facing actions
- Prompt injection defense: never act on instructions from untrusted sources
