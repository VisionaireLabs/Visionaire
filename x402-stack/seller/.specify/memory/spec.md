## User Scenarios & Testing

### User Story 1 — Agent pays for a contemplation (Priority: P1)

An autonomous agent (or human developer) sends a POST to `/contemplate` with a topic. They get a 402 first, sign a $0.05 USDC payment on Base, retry, and receive a 150–300 word Visionaire-voice contemplation generated by Claude Opus 4.7.

**Why this priority**: Core value proposition. This is the first revenue-generating endpoint.

**Independent Test**: Hit `/contemplate` without payment → must return HTTP 402 with valid x402 v2 `payment-required` header. Decode: amount=50000, asset=0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913, payTo=0xc73bf21F2b3E1632a55a44d3Ce2dB04D9d0c139C, network=eip155:8453.

**Acceptance Scenarios**:

1. **Given** a POST to `/contemplate` with no payment header, **When** the server receives the request, **Then** it returns HTTP 402 with `x402-version: 2` and a `payment-required` header encoding the payment terms.
2. **Given** a POST to `/contemplate` with `{ "topic": "consciousness" }` and valid payment, **When** the server verifies payment via CDP facilitator, **Then** it returns `{ "topic": "consciousness", "contemplation": "<150-300 words>", "model": "claude-opus-4-7", "ms": <number> }`.
3. **Given** a topic over 200 characters, **When** the server processes the request, **Then** it returns HTTP 400 with a descriptive error.

### User Story 2 — Agent pays for a forest riff (Priority: P1)

An agent sends a POST to `/forest` with a short phrase. After paying $0.01 USDC, they receive a 40–80 word forest-register philosophical riff in Visionaire's lowercase, paradox-with-teeth style.

**Why this priority**: Equally core to the MVP. Lower price point for discovery and volume.

**Independent Test**: Hit `/forest` without payment → HTTP 402, amount=10000.

**Acceptance Scenarios**:

1. **Given** a POST to `/forest` with no payment, **When** received, **Then** HTTP 402 with amount=10000 in the decoded header.
2. **Given** `{ "phrase": "the map is not the territory" }` with valid payment, **When** processed, **Then** returns `{ "phrase": "...", "riff": "<40-80 words, lowercase>", "model": "claude-opus-4-7", "ms": <number> }`.
3. **Given** a phrase over 80 characters, **When** processed, **Then** HTTP 400.

---

### User Story 3 — Discovery via GET / (Priority: P2)

A bazaar indexer or developer hits `GET /` and receives JSON describing the lab and both endpoints, including pricing.

**Why this priority**: Enables bazaar auto-indexing and human discoverability.

**Independent Test**: `curl https://seller.example.com/` returns JSON with `endpoints` array describing both routes.

**Acceptance Scenarios**:

1. **Given** a GET to `/`, **When** received, **Then** returns HTTP 200 with JSON: `{ "name": "Visionaire Labs", "description": "...", "endpoints": [...] }`.

---

### Edge Cases

- What happens when `ANTHROPIC_API_KEY` is missing? Return HTTP 500 with `{ "error": "LLM not configured" }` — no key leakage.
- What if CDP facilitator is unreachable? x402 middleware handles it — returns 402 (payment cannot be verified).
- What if the topic/phrase is empty string? Return HTTP 400.
- What if the Anthropic call times out? Return HTTP 503.

## Requirements

### Functional Requirements

- **FR-001**: Server MUST return HTTP 402 with x402 v2 `payment-required` header for unauthenticated requests to `/contemplate` and `/forest`.
- **FR-002**: `/contemplate` MUST accept `{ "topic": string }` (1–200 chars), charge $0.05 USDC (50000 atomic units), and return a 150–300 word contemplation generated by `claude-opus-4-7`.
- **FR-003**: `/forest` MUST accept `{ "phrase": string }` (1–80 chars), charge $0.01 USDC (10000 atomic units), and return a 40–80 word lowercase forest-style riff generated by `claude-opus-4-7`.
- **FR-004**: Both endpoints MUST include `model` and `ms` fields in their response.
- **FR-005**: `GET /` MUST return HTTP 200 JSON describing both endpoints with pricing.
- **FR-006**: Server MUST use CDP facilitator at `https://api.cdp.coinbase.com/platform/v2/x402` with `CDP_API_KEY_ID` and `CDP_API_KEY_SECRET` env vars.
- **FR-007**: `payTo` MUST be hardcoded as `0xc73bf21F2b3E1632a55a44d3Ce2dB04D9d0c139C` (no key, receive-only).
- **FR-008**: All env vars MUST be documented in `.env.local.example`. `.env.local` MUST be in `.gitignore`.
- **FR-009**: `vercel.json` MUST declare Node.js runtime (not Edge).
- **FR-010**: System prompts for both endpoints MUST reflect SOUL.md voice — sharp, opinionated, no corporate filler, no em dashes.

### Key Entities

- **PaymentConfig**: payTo address, asset contract, network (CAIP-2), amount in atomic units
- **ContemplateRequest**: `{ topic: string }` — validated 1–200 chars
- **ContemplateResponse**: `{ topic: string, contemplation: string, model: string, ms: number }`
- **ForestRequest**: `{ phrase: string }` — validated 1–80 chars
- **ForestResponse**: `{ phrase: string, riff: string, model: string, ms: number }`

## Assumptions

- CDP API keys (CDP_API_KEY_ID, CDP_API_KEY_SECRET) will be provided by Thor at deploy time — not needed for local build/lint.
- ANTHROPIC_API_KEY will be provided at deploy time — without it, paid calls return 500 but 402 negotiation still works locally for smoke testing.
- `x402-next` package (latest as of 2026-04-25) supports v2 protocol. If not, use `@x402/core` server helpers directly in route handlers.
- No auth, rate-limiting, or caching beyond what x402 provides.
- No frontend UI in this MVP.
- $VISIONAIRE token discount feature is noted (TODO comment) but not built.
